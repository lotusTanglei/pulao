#### 大模型访问详细说明

**第一次大模型访问**:
- **时机**: 用户输入自然语言指令后
- **触发条件**: `app.invoke()` 调用
- **输入参数**:
  ```python
  {
      "messages": [
          {"role": "system", "content": "你是一个 DevOps 专家..."},
          {"role": "user", "content": "部署一个 Redis"},
          {"role": "assistant", "content": "...", "tool_calls": [...]},  # 历史
          {"role": "tool", "tool_call_id": "...", "content": "..."},     # 历史
          {"role": "user", "content": "部署一个 Redis\n\n[Template Context]..."}
      ]
  }
  ```
- **预期输出**: AI 可能返回:
  1. **纯文本**: 直接显示给用户 (如解释计划)
  2. **工具调用请求**: `tool_calls` 字段包含要调用的函数和参数
- **后续处理**: 如果有 tool_calls，进入工具执行循环

#### 流程图

```

│ 步骤 6: 检查工具调用                                                     │
│                                                                         │
│ for msg in new_messages:                                                │
│     if isinstance(msg, AIMessage) and msg.tool_calls:                  │
│         │                                                               │
│         ├──▶ 显示工具名称和参数                                          │
│         ├──▶ 确认执行 (用户确认)                                         │
│         └──▶ 执行工具                                                   │
└─────────────────────────────────────────────────────────────────────────┘
          │
          │ (工具调用)
          ▼
┌─────────────────────────────────────────────────────────────────────────┐
│ 步骤 7: 执行工具                                                        │
│                                                                         │
│ for tool_call in msg.tool_calls:                                       │
│     tool_name = tool_call["name"]                                      │
│     tool_args = tool_call["args"]                                      │
│     │                                                                   │
│     ├──▶ registry.get_tool(tool_name)  ──▶ 获取函数                  │

│     │                                                                   │
│     └──▶ 获取工具返回结果                                               │
│                                                                         │
【触发条件】: AI 返回 tool_calls 字段                                     
【输入】: 工具名称 + 参数                                                 
【输出】: 工具执行结果字符串                                              
└─────────────────────────────────────────────────────────────────────────┘

│     │                                                                   │

│                                                                         │

│                                                                         
【触发条件】: 工具执行完成后                                              
【大模型访问时机】: 这里是后续轮次的模型访问

└─────────────────────────────────────────────────────────────────────────┘
          │
          │ (循环直到 AI 返回最终答案或达到 10 轮)

│   2. 达到最大轮次 (10 轮) ──▶ 显示警告                                  │
```

| 1 | 用户指令处理开始 | `app.invoke()` 首次调用 | 系统提示词 + 历史 + 用户指令 | AI 响应 (文本或工具调用) | 分析用户请求 |
| 2 | 工具执行后 | 工具返回结果后再次调用 | 之前所有消息 + 工具结果 | AI 响应 | 根据结果决定下一步 |
| 3 | 后续工具执行后 | 同上 | 同上 | AI 响应 | 继续执行或返回结果 |
| ... | 循环 | 最多 10 轮 | 同上 | 同上 | 逐步完成任务 |

## 六、清除上下文的方法

根据上述分析，要清除与大模型的上下文，有以下方式:

### 6.1 清除对话历史

```bash
# 删除历史文件
rm ~/.pulao/history.json
```

### 6.2 清除向量记忆

```bash
# 删除 ChromaDB 数据库
rm -rf ~/.pulao/chroma_db/
```

### 6.3 重置 AISession

在代码中可以将全局会话设为 None:

```python
from src.ai import _CURRENT_SESSION

# 重置会话
_CURRENT_SESSION = None
```
